# Phase 1.5：Financial Market News Collector 反馈 Web 设计文档

> 版本：v1.0 ｜ 日期：2026-06-28 ｜ 依赖：Phase 1（子计划 1–6）已完成
> 用途：在现有 Horizon fork 仓库中新增一个轻量 Web，让第一批用户在浏览器看 brief，并收集使用数据与反馈。**不商业化、不做账户体系**。

## 1. 目标与边界

**目标**：给第一批用户一个能用 Safari / 任意浏览器打开的网页，方便阅读每日 brief；同时采集**行为数据（点击/浏览）**与**主动反馈（👍/👎 + 评论）**，用于打磨 view 质量与产品方向。

**明确不做（留给 Phase 2）**：用户账户/登录、个性化订阅、付费、多租户。本阶段一切匿名。

**关键事实**：现有 GitHub Pages 是静态站，无法接收反馈/记录行为。本阶段新增一个**带小后端的动态 Web**（FastAPI），读写同一个 `data/marketnews.db`。`event`、`feedback` 两张表在 Phase 1 已预留，本阶段启用。

## 2. 品牌与视觉规范（Cartesian / Playfair）

- **品牌名**：Financial Market News Collector
- **字体**：标题 `Playfair Display`（衬线，Google Fonts）；正文/UI `Work Sans`
- **配色（暖中性，近单色）**：
  ```
  --bg:        #f3f1ea   页面背景（暖纸色）
  --card:      #faf8f2   卡片/ view 块
  --ink:       #1a1a1a   主文字 / 细分隔线
  --muted:     #6b665d   次要文字
  --faint:     #8a8275   更弱（日期/反馈计数）
  --hair:      #d9d4c8   发丝线 / 边框
  --chip:      #e9e5da   标签底色
  ```
- **风格**：大量留白、发丝细线、近单色、克制墨点强调；响应式（手机 Safari 也好看）。
- **页头**：仅**刊名 + 日期**（不显示 "DAILY MARKET BRIEF" 小标、不显示 "每日刊"、**不显示抓取/精选数量**等内部统计）。
- **页尾**：固定免责声明。

## 3. 页面与信息结构

**每条新闻三层结构**（缺一不可）：
1. **标题** headline
2. **内容概括** summary —— 一两句客观陈述这条新闻说了什么
3. **中性影响 view** —— "影响（中性）：…"（合规：无买卖/方向/目标价）

外加：`[原文 ↗]` 链接、分类标签（行业组/产品组/地区）、👍/👎 + 评论入口。

**页面**：
- `综合市场观点`（整期 market_view_text，Playfair 斜体引文）
- 分类筛选 chips（全部 / 行业组 / 地区 / 一级二级），点击筛当前 brief 的条目
- `今日要闻`（按 rank 排序的 brief_item 列表）

**内容概括来源（定稿：方案②）**：由 DeepSeek 在**生成 view 的同一次调用里顺带产出**一句中文客观概括（near-zero 额外 token、一致中文化），写入 `brief_item.summary`。需对子计划 5 的 `ViewGenerator` 做小扩展：让其同时返回 `summary` 与 `view_text`；`BriefAssembler` 写入两者。

## 4. 数据采集

- **行为埋点（event 表）**：
  - `view`：某条 brief_item 进入视口（前端 IntersectionObserver 上报一次）
  - `click`：用户点击该条的 `[原文]` 链接
- **反馈（feedback 表）**：每条 👍/👎（rating: `up`/`down`）+ 可选评论 comment。
- **匿名标识**：浏览器 `localStorage` 生成并保存一个 UUID（`anon_id`），随 event/feedback 上报，仅用于"同一浏览器去重计数"，**不收集任何身份信息**。
- 需要给 `event`、`feedback` 两表各加一列 `anon_id TEXT`（迁移：`ALTER TABLE ... ADD COLUMN`，幂等）；`user_id` 保持可空（匿名时为 NULL）。

## 5. 技术架构

```
现有管线（子计划1-6）每日生成 brief 入库 (data/marketnews.db)
                    │  (读)
        ┌───────────▼─────────────┐
        │  FastAPI Web (新增 src/web/) │
        │  - GET /            最新一期 │
        │  - GET /brief/{date}  指定日期│
        │  - GET /review        跨期回顾│
        │  - POST /api/feedback        │  (写 feedback)
        │  - POST /api/event           │  (写 event)
        │  Jinja2 模板 (Cartesian 视觉) │
        └──────────────────────────────┘
                    │
              用户浏览器 (Safari/手机)
```

- **后端**：FastAPI（与现有 Python 栈一致），同步读写 SQLite（复用 Phase 1 的 `Database`/Repos）。
- **模板**：Jinja2 服务端渲染；CSS 内联本设计系统；Google Fonts 引 Playfair + Work Sans。
- **前端 JS**：极少量——管理 `anon_id`、点赞/踩、评论提交、原文点击上报、IntersectionObserver 浏览上报、分类筛选（客户端隐藏/显示）。
- **新增 Repos**：`EventRepo`、`FeedbackRepo`；新增渲染查询 `WebQuery.brief_for_web(date)`（取某期 brief + 每条的 tags + 主原文 url）。

## 6. 部署（让第一批用户能访问）

- 推荐 **Render**（免费档）：Web Service 跑 FastAPI + **持久化磁盘**放 `marketnews.db`；用 Render Cron Job 每日跑 `uv run horizon --marketnews` 生成新 brief。一处部署，用户拿到公开 URL。
- SQLite 必须放在持久化磁盘（否则重新部署丢数据）。
- 备选：Railway / Fly.io 同理；本地自测用 `uvicorn`。

## 7. 验收（DoD）

- 浏览器打开公开 URL 能看到当日 brief（Cartesian 视觉、每条三层结构、原文链接、免责声明）。
- 点 👍/👎、写评论能成功入 `feedback`；点原文、浏览条目能入 `event`（带 anon_id）。
- 能按行业组/地区/一级二级筛选当期；能翻看历史日期。
- 匿名、无需登录；手机浏览器显示正常。
- 新增后端逻辑有测试；不破坏 Phase 1 的 322 个测试。
