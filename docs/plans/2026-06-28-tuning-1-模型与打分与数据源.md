## B. 去掉 view 里冗余的"中性影响说明"标题

**问题**：每条 view 输出都带一个多余前缀（"中性影响说明""中立影响说明""中性影响笔记""Neutral Impact Note"），而页面/Markdown 本来就有 `影响（中性）：` 标签，重复且不统一。

**改法**（`src/view/view_templates.py`）：在 `NEUTRAL_SYSTEM` 和各 user 模板里明确要求——
- 返回 JSON 的 `view` 字段**只放中性影响内容本身**，**不得**包含任何标题、前缀、"中性影响说明/笔记/Note"之类的字样，也不要 Markdown 小标题。
- `summary` 字段同理，只放一句客观概括。

可在 system prompt 末尾加一句硬约束，例如：
`The "view" field must contain ONLY the neutral impact text itself — no heading, no prefix, no labels like "中性影响说明" or "Neutral Impact Note". Do not start with a title.`

- 更新/新增一个测试：断言生成的 view 文本不以"中性影响"/"Neutral Impact"等字样开头（可在 `tests/view/` 加一个针对清洗后输出的断言；若用假 AI，则改造假 AI 返回带前缀的内容、验证被规整掉，或直接在 ViewGenerator 加一步 `_strip_redundant_heading()` 并测试它）。
- 建议实现一个小工具函数 `_strip_redundant_heading(text)`：去掉开头形如 `**中性影响说明**`、`中立影响说明：`、`Neutral Impact Note` 等前缀行，保证稳健（不只靠提示词）。
- commit: `fix(view): strip redundant neutral-impact heading from view text`

## C. 打分给并购/融资/交易/行业类加权（提升 IB 相关内容占比）

**问题**：首跑选题被地缘政治刷屏，M&A / 一级市场 / 二级市场 / 行业类内容偏少。

**改法**（`src/ai/financial_prompts.py` 的 `FINANCIAL_SCORE_SYSTEM`）：在打分指引里**显式拔高投行相关事件的重要性**，例如新增一段：

```
Prioritization (boost importance for investment-banking-relevant events):
- M&A / takeovers / asset sales, ECM (IPO/follow-on), DCM (bond issuance),
  leveraged finance, restructuring → treat as HIGH importance (typically 7-9).
- Private-market financings (VC rounds, PE buyouts, fund closings, pre-IPO,
  private placements) → HIGH importance (7-9).
- Sector/industry coverage developments (earnings with read-through, guidance,
  regulatory/antitrust, major product/capacity moves) → MEDIUM-HIGH (6-8).
- Pure geopolitics / general macro with no direct corporate or market mechanism
  → keep MODERATE unless it is clearly market-moving for specific assets.
Deals, financings, and sector developments should generally rank above generic
geopolitical headlines of similar prominence.
```

- 保持中性原则不变（仍不输出建议/方向/目标价）。
- 现有打分测试用假 AI，不受影响；如有断言具体分值的测试需相应调整。
- commit: `feat(scoring): weight M&A / financings / sector developments higher`
