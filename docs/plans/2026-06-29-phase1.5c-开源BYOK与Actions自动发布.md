> 核心原则：**API key 永远只存在于运行环境(.env 或 GitHub Secret)，绝不进代码/仓库。**

## A. 开源可复用（BYOK，别人 clone 后自带 key 运行）

目标：任何人 `git clone` 后，填自己的 key，就能本地跑出网页。基本已具备，主要是补文档与一个导出命令。

**README 增加「快速开始」段（中英双语可选）：**
```bash
git clone <repo> && cd <repo>
uv sync --extra dev
cp .env.example .env        # 然后编辑 .env，填入你自己的 DEEPSEEK_API_KEY
cp data/config.example.json data/config.json
uv run horizon --marketnews --hours 24      # 抓取+生成今日 brief 入库
uv run python -m src.web.static_export      # 导出静态看板到 site/index.html
# 直接用浏览器打开 site/index.html；或 uv run python -m src.web.run 看交互版
```

**确认 `.env.example` 含且仅含占位**：`DEEPSEEK_API_KEY=`（无真值），且 `.gitignore` 已忽略 `.env`（已具备）。

## B. 一键静态导出脚本 `src/web/static_export.py`（含历史 + 分类 + 公司回顾）

把静态导出做成**客户端数据驱动的看板**：导出 `site/data.json`（全部历史 brief 数据）+ `site/index.html`（纯前端 App，读 data.json 渲染），从而在**纯静态、零服务器**下保留历史回顾、分类回顾、公司回顾。

**接口**：`python -m src.web.static_export [--out site]`
- 导出**全部历史** brief（不是只导最新一期）。

**B1. `data.json` 结构**（从 DB 导出全部 brief + items + tags + entities；view 经 `_clean_view` 清洗）：
```json
{
  "generated_at": "2026-06-29T...",
  "taxonomy": [{"dim":"industry_group","code":"TMT","label":"TMT"}, ...],
  "entities": [{"id":12,"name":"Acme","count":3}, ...],
  "briefs": [
    {"date":"2026-06-29","market_view":"...","items":[
      {"rank":1,"headline":"...","summary":"...","view":"...","url":"...",
       "tags":[{"dim":"product_group","code":"MA","label":"M&A/Advisory"}],
       "entities":[{"id":12,"name":"Acme"}]}
    ]}
  ]
}
```
新增 `WebQuery` 方法：`export_all() -> dict`（聚合所有日期的 brief_for_web + 实体清单 + taxonomy 选项）。实体来自 `entity`/`story_entity`，含每实体被提及次数。

**B2. `index.html`（Cartesian 视觉 + 纯前端 JS）**，加载 `data.json` 后支持：
- **历史**：日期下拉/列表，切换查看任意一天的 brief（默认最新一天）。
- **分类筛选**：按 market_type / industry_group / product_group / region / asset_class 的 chips 过滤当前视图（支持「仅当天」与「全部历史」两种范围切换）。
- **公司回顾**：点任意标签/实体名 → 列出**所有历史里提到该公司的条目**（按日期倒序），实现单公司纵向回顾。
- 每条仍是三层结构（标题 / 概括 / 影响）+ 原文链接 + 标签；**无赞/踩/评论**。
- 底部固定：免责声明 + `Feel free to contact me at 1079919402@qq.com`。
- 自包含：CSS 内联、字体 Google Fonts CDN、图标 jsdelivr CDN；仅额外依赖同目录 `data.json`。

**实现要点**：JS 原生即可（无框架）；所有渲染/筛选/回顾在浏览器端完成；HTML/文本转义防注入。

**测试** `tests/web/test_static_export.py`：建 2 期 brief（不同日期、含 tags + entity，且 1 条带 JSON 残渣），调导出，断言：
- 生成 `data.json` 含两期、taxonomy、entities；某条的 view 已清洗（不含 `"view":`）。
- `index.html` 含刊名、`影响：`占位渲染逻辑、`1079919402@qq.com`、免责声明，且**不含** `data-vote`（无赞踩）。
- `data.json` 里能按 entity 找到跨期条目（验证公司回顾数据齐全）。

commit: `feat(web): static dashboard with history / taxonomy / company review (data.json + client app)`

**B3. 把导出接进管线（一条命令搞定）**：在 `src/main.py` 的 `--marketnews` 流程里，`MarketNewsOrchestrator.run()` 完成后**自动调用一次静态导出**写 `site/`。这样本地 `uv run horizon --marketnews` 一条命令即可「生成 brief + 导出静态看板」；Actions 里导出步骤可省略或保留。commit: `feat(web): auto-export static dashboard after pipeline run`

## C. GitHub Actions 定时自动发布（你自己的部署）

**新建 `.github/workflows/daily-publish.yml`**（每天北京 7 点 = UTC 23:00）：

```yaml
name: Daily brief publish
on:
  schedule:
    - cron: "0 23 * * *"   # 北京时间 07:00
  workflow_dispatch: {}      # 也允许手动触发
permissions:
  contents: write            # 提交回库
  pages: write
  id-token: write
concurrency:
  group: daily-publish
  cancel-in-progress: false
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install uv
        run: curl -LsSf https://astral.sh/uv/install.sh | sh
      - name: Sync deps
        run: ~/.local/bin/uv sync
      - name: Prepare config
        run: cp data/config.example.json data/config.json
      - name: Run pipeline (generate brief)
        env:
          DEEPSEEK_API_KEY: ${{ secrets.DEEPSEEK_API_KEY }}
        run: ~/.local/bin/uv run horizon --marketnews --hours 24
      - name: Export static dashboard
        run: ~/.local/bin/uv run python -m src.web.static_export --out site
      - name: Commit updated DB + site (persist dedup/history)
        run: |
          git config user.name "github-actions"
          git config user.email "actions@github.com"
          git add data/marketnews.db site/ docs/ 2>/dev/null || true
          git commit -m "chore: daily brief $(date -u +%F)" || echo "no changes"
          git push || echo "nothing to push"
      - uses: actions/upload-pages-artifact@v3
        with:
          path: site
  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
    steps:
      - uses: actions/deploy-pages@v4
```

**要点 / 注意**：
- **持久化**：把 `data/marketnews.db` 提交回库，跨天去重与历史才连续（DB 不大，可接受）。若不希望 db 进库，可改用 `actions/cache`，但提交最简单。
- **DeepSeek key**：只来自 `secrets.DEEPSEEK_API_KEY`，不进代码。
- **慢的问题**：管线在 Actions 云端后台跑，慢不影响你；但建议先做 `tuning-2`（并发/批量）确保在 Actions 时限内跑完。
- **首次需手动触发一次**（Actions 页面点 Run workflow）确认通路。

## D. 用户一次性配置（你在 GitHub 网页操作）

1. 把仓库推到 GitHub（私有/公开均可）。
2. `Settings → Secrets and variables → Actions → New repository secret`：名 `DEEPSEEK_API_KEY`，值填你的 key。
3. `Settings → Pages → Build and deployment → Source` 选 **GitHub Actions**。
4. 到 `Actions` 页手动跑一次 `Daily brief publish` 验证；成功后网址为 `https://<用户名>.github.io/<仓库名>/`。
5. 之后每天北京 7 点自动更新，无需电脑开机。

## E. 开源前安全检查（重要）

公开仓库前，确认 git 历史里从没误提交过 key/.env：
```bash
git log --all --full-history -- .env            # 应无输出
git grep -i "sk-" $(git rev-list --all) 2>/dev/null | head   # 扫历史里的疑似 key，应无
```
若发现历史里有 key：在 DeepSeek 后台**作废轮换**该 key，并用 `git filter-repo` 清历史（或重建仓库）。`.env` 已被 `.gitignore`，正常情况下是干净的。
